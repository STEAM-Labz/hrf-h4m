Introduction

Welcome to the comprehensive knowledge base for STEAM-Labz's HackRF One and H4M Portapack - your definitive guide to mastering these powerful Software Defined Radio (SDR) tools. Whether you're a beginner or an experienced RF engineer, this knowledge base provides everything you need to understand, configure, and utilize these devices to their full potential.

The HackRF One is an open-source hardware platform that can be used as a USB peripheral or programmed for stand-alone operation. It's capable of transmission or reception of radio signals from 1 MHz to 6 GHz, making it an incredibly versatile tool for RF exploration and development.

The H4M Portapack extends the HackRF One with a touchscreen interface, controls, and additional features that transform it into a portable, standalone device for signal analysis and transmission - no computer required.

Setup Instructions

SD Card Setup

  1. Purchase a good quality microSD card (32GB or smaller recommended) from a reputable brand like SanDisk or Samsung
  2. Format the card as FAT32 (required for Mayhem firmware versions prior to 2.1.0)
  3. Download the latest firmware release from the Mayhem GitHub releases page
  4. Extract the "Mayhem_vX.X.X_copy_to_SD_card.zip" file to your SD card
  5. Insert the SD card into your H4M Portapack

Starting with Mayhem 2.1.0, exFAT formatted SD cards are also supported, allowing for cards larger than 32GB.

Firmware Installation

  1. Turn on your Portapack by sliding the power switch up
  2. Navigate to "Utilities" and select "Flash Utility"
  3. Select the file named "portapack-h1_h2-mayhem.bin" from the list
  4. Confirm the installation and wait for the flashing process to complete
  5. Your Portapack will restart with the new firmware installed

Alternatively, you can use the online updater tool (not available on Mac) which updates both the firmware and SD card files automatically.

Basic Configuration

  1. Calibrate the touch screen: Go to Settings → Calibration (hold down on the markers, don't just tap them)
  2. Set date and time: Go to Settings → Date & Time
  3. Customize user interface: Go to Settings → User Interface
    • Enable "Show splash" and "Back button in menu"
    • Hide unnecessary status bar icons
    • Set color theme (green is popular)
  4. Save settings: Go to Settings → P.Memory Management to save your settings to the SD card
  5. Set custom splash image: Place 240x304 pixel 24-bit BMP files in the "SPLASH" folder on your SD card, then use Utilities → File Manager to select one

Battery and Charging

  • The H4M uses a new power design with a physical on/off switch that completely disconnects the battery
  • To view battery info, tap the battery icon in the status bar
  • When charging via USB, make sure the device is powered ON (switch in the up position)
  • You can choose between "IC" or "Volt" battery monitoring methods - "Volt" is generally more reliable
  • Try to keep the battery voltage above 3.0V for optimal battery life

Firmware Updates

Mayhem Firmware

The Mayhem firmware is the most popular and feature-rich custom firmware for the HackRF Portapack. It adds numerous applications and improvements over the stock firmware.

Update Methods

Method 1: Direct Update (Recommended)
  1. Download the latest release from GitHub
  2. Extract the SD card files to your microSD card
  3. Insert the card into your Portapack
  4. Go to Utilities → Flash Utility and select the firmware file
Method 2: Web Updater
  1. Visit https://hackrf.app
  2. Connect your Portapack to your computer via USB
  3. Follow the on-screen instructions to update
  4. This method automatically updates both firmware and SD card files

Note: This method doesn't work on macOS.

Method 3: DFU Mode (Recovery)

If your device won't boot or is stuck, you can use DFU mode to recover:

  1. Power off your device
  2. Hold the DFU button (small button on the HackRF board)
  3. Connect to USB while holding the button
  4. Use dfu-util command line tool or HackRF.app to flash the firmware

Firmware and Applications Compatibility

The applications on the SD card will only work with the exact firmware version they were compiled for. Always update both the firmware and SD card files together to ensure compatibility.

Common Firmware Issues

Device not booting after update

If your device doesn't boot after a firmware update:

  1. Try removing and reinserting the SD card
  2. Ensure your battery is charged
  3. If still not working, try a DFU recovery flash
Applications not appearing or crashing

If applications are missing or causing crashes:

  1. Make sure you're using the SD card files that match your firmware version
  2. Try reformatting your SD card and copying the files fresh
  3. Check that your SD card is not counterfeit
SD card not recognized

If your SD card is not being recognized:

  1. Ensure the card is properly formatted (FAT32 for older firmware, FAT32 or exFAT for 2.1.0+)
  2. Try a different, known good SD card
  3. Check if the SD card slot is damaged or has debris

Tutorials

Getting Started with Basic Applications

Audio Application

The Audio application allows you to listen to radio signals across the HackRF's frequency range:

  1. From the main menu, go to Receive → Audio
  2. Select the modulation type in the top left (WFM for FM radio, NFM for narrow FM, AM for amplitude modulation)
  3. If listening to FM broadcast stations, select WFM and set frequency to 88-108 MHz
  4. Adjust the LNA, VGA, and AMP settings to optimize reception
  5. Set volume and squelch levels as needed
  6. Use the tuning knob to scan through frequencies or directly enter a known frequency

Tip: For FM broadcast stations, try 88-108 MHz. For ham radio, try 144-148 MHz with NFM. For shortwave, try 5-30 MHz with AM mode.

Looking Glass Application

Looking Glass is a spectrum analyzer showing a waterfall display of radio activity:

  1. From the main menu, go to Receive → Looking Glass
  2. Select a frequency range preset or set custom min/max frequencies
  3. Adjust LNA and VGA gain to see signals clearly
  4. Active signals will appear as bright lines on the waterfall display
  5. Use the marker to identify the exact frequency of a signal
  6. Press the center button to open the Audio app on the selected frequency

Note: The bright vertical line in the center is a "DC spike" - this is normal and not an actual signal.

Signal Capture and Replay

Capture and replay RF signals like remote controls:

  1. First use the Search app to find the signal's frequency:
    • Go to Receive → Search
    • Set the frequency range (e.g., 433-435 MHz for many remotes)
    • Press the remote button and note the detected frequency
  2. Capture the signal:
    • Go to Receive → Capture
    • Enter the frequency found earlier
    • Click the red record button and activate the remote
    • Stop recording when complete
  3. Replay the signal:
    • Go to Transmit → Replay
    • Load your recording
    • Press the green arrow to transmit
Important: Always use the appropriate antenna for the frequency you're transmitting on to avoid damaging your device.

Advanced Applications

ADS-B Aircraft Tracking

Track aircraft in your vicinity by receiving their ADS-B transmissions:

  1. Go to Receive → ADS-B
  2. Adjust LNA and VGA gain settings (increase for better range, but avoid saturation)
  3. The application will automatically tune to 1090 MHz and decode aircraft signals
  4. Aircraft will appear in the list with their callsign, altitude, and other information
  5. Select an aircraft to see more details
  6. Switch to Map view to see aircraft positions plotted on the map

Tip: For best results, use an antenna with good 1090 MHz reception and position it with clear view of the sky.

Radio Signal Analysis

Analyze signals from keyfobs, wireless sensors, and other devices:

  1. For key fobs and remotes:
    • Go to Receive → Capture or Receive → Search to find the frequency
    • Most operate in common ISM bands (315, 433, 868, or 915 MHz)
    • Use Spectrum view to see the signal pattern
  2. For TPMS (Tire Pressure Monitoring System):
    • Go to Receive → TPMS
    • The app automatically scans common TPMS frequencies
    • When near vehicles, it will display tire pressure and temperature data
  3. For digital signals:
    • Use the appropriate decoder app (NRF, AFSK, etc.)
    • Adjust the frequency and gain settings
    • The app will attempt to decode and display the data

Custom Applications Development

For advanced users interested in developing custom applications:

  1. Clone the Mayhem firmware repository:
    git clone https://github.com/portapack-mayhem/mayhem-firmware.git
  2. Set up the development environment by following the instructions in the documentation
  3. Study existing applications in the /firmware/application folder
  4. Create your application using C++ and the provided API
  5. Test your application using the provided simulator
  6. Compile and flash to your device

For more detailed instructions, refer to the development guide in the Mayhem GitHub wiki.

Frequently Asked Questions

What's the difference between HackRF One and the H4M Portapack?

The HackRF One is a standalone SDR board that requires a computer to operate. The H4M Portapack is an add-on that attaches to the HackRF One, providing an LCD screen, controls, and battery power, allowing it to function as a portable standalone device without needing a computer.

The H4M specifically is the latest version of the Portapack with several improvements over previous versions including a better screen, power management, GPIO port, and more.

What is the range of the HackRF/Portapack?

The reception range depends on many factors:

  • The frequency being monitored
  • The type and quality of the antenna being used
  • Environmental conditions and obstructions
  • The power level of the transmitting device

As a general rule, VHF/UHF signals (above 30 MHz) are primarily line-of-sight, so reception is limited to about 30-40 km maximum in open terrain. HF signals (below 30 MHz) can travel much further due to ionospheric reflection.

When transmitting, the HackRF has relatively low power output (10-20 mW), limiting its effective range to short distances.

Is it legal to use the HackRF/Portapack?

The HackRF and Portapack are legal to own and use in most countries. However, how you use them is subject to radio regulations:

  • Receiving signals is generally legal (with some exceptions for encrypted or protected services)
  • Transmitting without proper authorization is illegal in most jurisdictions
  • Jamming or interfering with any communications is illegal almost everywhere
  • Reproducing certain protected signals (like key fobs) may be illegal in some areas

Always research and follow your local radio regulations. In the US, this would be FCC regulations; other countries have their own equivalent agencies.

Which antennas should I use with my HackRF?

For best performance, use antennas designed for the specific frequency range you're interested in:

  • The telescopic antenna can work for general reception but is not ideal
  • For 2.4 GHz (Wi-Fi, Bluetooth): Use the included blade antenna
  • For cellular frequencies: Use the GSM/3G/4G antenna
  • For HF bands (below 30 MHz): Use a longer wire antenna (5+ meters)
  • For specific applications like ADS-B or TPMS: Consider purpose-built antennas

Important: When transmitting, always use an antenna designed for the frequency you're transmitting on to avoid damaging the HackRF.

How long does the battery last on the H4M Portapack?

Battery life depends on your usage patterns:

  • Typical usage (mostly receiving): 3-4 hours
  • Heavy transmission use: 1.5-2.5 hours
  • Standby with display on: 4-5 hours

To maximize battery life:

  • Lower the screen brightness
  • Avoid constant transmission
  • Turn off the device when not in use (the H4M's power switch completely cuts power)
  • Keep the battery charged above 3.0V to maintain battery health
Can the HackRF/Portapack intercept cell phone calls or text messages?

Modern cellular communications use advanced encryption and digital protocols that the HackRF cannot easily decode. While it can receive the raw signals in cellular frequency bands, it doesn't have the capability to:

  • Decode encrypted cellular communications
  • Intercept calls or messages from modern networks (4G/5G)
  • Clone or spoof modern cellular devices

Additionally, attempting to intercept cellular communications is illegal in most jurisdictions and can result in serious penalties.

My HackRF isn't detecting any signals. What's wrong?

Several factors can cause poor signal reception:

  1. Incorrect frequency - Make sure you're on the right frequency band
  2. Gain settings too low - Try increasing LNA and VGA gain
  3. Improper antenna - Use an antenna appropriate for the frequency
  4. Environmental obstacles - Move to a better location with fewer obstructions
  5. Damaged LNA - The 14dB LNA on HackRF can be damaged by static electricity; try toggling it on/off
  6. Firmware issues - Update to the latest firmware

For RF sensitivity testing, try tuning to local FM broadcast stations (88-108 MHz) as a first test, as these are typically strong enough to receive even with non-optimal setups.

Do I need a HAM radio license to use the HackRF?

For receiving only, you generally don't need any license. For transmitting:

  • A HAM (amateur radio) license is required to transmit on amateur radio frequencies
  • Some frequencies (ISM bands) allow unlicensed transmission with specific power limitations
  • Other frequencies require specific licenses or are restricted entirely

Getting a HAM radio license is recommended if you plan to do any significant transmitting with your HackRF. The license exam teaches important concepts about responsible radio use and regulations.

How does the H4M compare to previous Portapack versions?

The H4M offers several significant improvements over previous Portapack versions (like the H2):

  • Proper power on/off switch that completely cuts power (eliminates battery drain when off)
  • Matte screen for better visibility outdoors
  • Advanced battery management system with detailed battery information
  • Faster charging via USB-C
  • Built-in microphone with toggle switch between internal and external mic
  • Automatic audio switching between headphones and speaker
  • Redesigned navigation wheel that's more durable
  • Added GPIO port for custom add-ons and extensions
